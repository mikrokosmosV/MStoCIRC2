# MStoCIRC2 Output Reference

This document summarizes the output files and output directories generated by `MStoCIRC2` across `orf`, `search`, `eval`, `dea`, `nonquant`, and `quant` usage scenarios.

## 1. Output Directory Layout

### 1.1 When `-o/--output-dir` is provided

MStoCIRC2 writes results directly into the user-specified directory.

### 1.2 When `-o/--output-dir` is omitted

MStoCIRC2 automatically creates a new run directory under the current working directory:

- `MStoCIRC2_orf_YY-MM-DD.N`
- `MStoCIRC2_search_YY-MM-DD.N`
- `MStoCIRC2_eval_YY-MM-DD.N`
- `MStoCIRC2_dea_YY-MM-DD.N`
- `MStoCIRC2_nonquant_YY-MM-DD.N`
- `MStoCIRC2_quant_YY-MM-DD.N`

Here `N` is the per-day incrementing index for the same module name, such as `.1`, `.2`, `.3`, and so on.

### 1.3 Integrated workflow layout

For integrated commands, MStoCIRC2 creates stage-specific subdirectories under the root output directory:

- `mstocirc2 nonquant`: `orf/`, `search/`, `eval/`
- `mstocirc2 quant`: `orf/`, `search/`, `eval/`, `dea/`

This means an integrated quantitative run typically looks like:

```text
MStoCIRC2_quant_YY-MM-DD.N/
├── orf/
├── search/
├── eval/
└── dea/
```

In addition, integrated `quant` may create hidden helper files under:

```text
.mstocirc2_runtime/dea/
```

These files are runtime-generated auxiliary inputs used to reconstruct DEA design tables from the manifest or from `experiment_annotation.tsv`.

## 2. ORF Module Outputs

Default location:

- direct `orf` run: the ORF output directory
- integrated `nonquant`/`quant`: `<root>/orf/`

### 2.1 Core FASTA and mapping files

| File or pattern | Description |
| --- | --- |
| `circRNA_full_length.fasta` | Full-length mature circRNA nucleotide FASTA exported by the ORF stage and used as the sequence foundation for downstream translation evaluation. |
| `circRNA_all_ORF.fasta` | Raw translated ORF amino-acid FASTA before final deduplication restoration and final export filtering. |
| `circRNA_bsj_corf.fasta` | Final amino-acid FASTA of BSJ-aware circRNA ORFs that is consumed by downstream `search` and `eval` modules. |
| `circRNA_bsj_corf_nt.fasta` | Nucleotide-sequence counterpart of `circRNA_bsj_corf.fasta`, preserving the final ORF set in nucleotide form. |
| `circRNA_ORF_mapping.tsv` | Mapping table linking each exported final ORF to its source ORF header, source amino-acid sequence, and source nucleotide sequence. |

### 2.2 Conditional annotation file

| File or pattern | Description |
| --- | --- |
| `circ_info_omit.gff` | Genome-mode-only annotation summary retained for downstream gene/strand annotation during `eval`; it is not kept for pure sequence-input ORF runs. |

## 3. Search Module Outputs

Default location:

- direct `search` run: the search output directory
- integrated `nonquant`/`quant`: `<root>/search/`

### 3.1 MStoCIRC2-generated stable files

| File or pattern | Description |
| --- | --- |
| `decoys-contam-database.fasta` | Combined target-decoy FASTA assembled from circRNA ORFs and canonical proteins for FragPipe-based searching. |
| `fragpipe_input.fp-manifest` | FragPipe-normalized manifest regenerated from the user manifest so that headless FragPipe receives a valid four-column input table. |
| `run.workflow` | Workflow template copied and rewritten with the absolute database path for the current run. |
| `fragpipe.run.log` | Live headless FragPipe execution log capturing the launch command and streamed runtime output. |

### 3.2 FragPipe-native result files

FragPipe itself writes many additional files into the search output directory, and the exact set depends on workflow type and acquisition mode.

Typical examples include:

- `combined_peptide.tsv`
- `peptide.tsv`
- `dia-quant-output/report.pr_matrix.tsv`
- `experiment_annotation.tsv`
- other FragPipe workflow-native reports, logs, and intermediate products

These files are generated by FragPipe rather than by MStoCIRC2's own Python code, so their presence depends on the selected workflow template, data type, and enabled search engines.

## 4. Evaluation Module Outputs

Default location:

- direct `eval` run: the evaluation output directory
- integrated `nonquant`/`quant`: `<root>/eval/`

### 4.1 MS parsing and peptide filtering files

| File or pattern | Description |
| --- | --- |
| `tsv_result.txt` | Initial circRNA-linked peptide evidence table parsed from the supplied MS input after identifier normalization and peptide cleaning. |
| `tsv_result_remove_line.txt` | Filtered peptide-evidence table after removing peptides explainable by the canonical linear proteome. |
| `peptide_line.txt` | Flat peptide list retained after linear-background filtering and used by later mapping and scoring steps. |
| `diamond_peptides_query.fasta` | Temporary FASTA generated from retained peptide candidates for `diamond blastp` filtering against the canonical proteome. |
| `eval_diamond_protein_db.dmnd` | `diamond` database built from the canonical protein FASTA for evaluation-stage peptide filtering. |
| `diamond_filter_result.txt` | Raw `diamond blastp` result file used to blacklist peptides with linear-proteome support. |

### 4.2 Optional IRES and m6A prediction files

| File or pattern | Description |
| --- | --- |
| `all_circRNA_ires.fasta` | FASTA of ORF-linked circRNA sequence windows submitted to DeepCIP for IRES-related translation-potential scoring. |
| `all_circRNA_m6a_windows.fasta` | FASTA of sequence windows submitted to bundled DeepCircM6A inference for m6A site prediction. |
| `all_circRNA_m6a_pred.txt` | DeepCircM6A prediction output summarizing m6A-positive windows detected for candidate ORFs. |
| `deepcircm6a.run.log` | Run log written when DeepCircM6A inference is invoked, recording the executed command and captured stdout/stderr summary. |
| `deepcircm6a.stderr.log` | Error log written only when DeepCircM6A exits abnormally or fails to generate its expected prediction output. |

### 4.3 Core evaluation result

| File or pattern | Description |
| --- | --- |
| `circ_predict.txt` | Ranked circRNA translation-potential result table integrating peptide evidence, BSJ-crossing support, IRES prediction, m6A prediction, and a composite coding-function score. |

### 4.4 Per-sample and visualization outputs

| File or pattern | Description |
| --- | --- |
| `ms_sample_orf_identification.txt` | Per-MS-sample summary table listing how many circRNA ORFs are supported in each sample and which ORFs are identified. |
| `Peptide_coverage_map/*_coverage.pdf` | Top-ranked ORF peptide-coverage plots showing where retained peptides map onto the translated cORF sequence. |
| `circRNA_ORF_map/*_circ.pdf` | Top-ranked circular ORF structure plots showing ORF location, BSJ position, and peptide-supported regions on the circular transcript map. |

## 5. Detailed Field Guide for `circ_predict.txt`

`circ_predict.txt` is the main reference table exported by `eval`, and it is also the default circRNA reference consumed by the `dea` module.

### 5.1 Column list

The exact header depends on whether strand annotation is available.

Common columns:

1. `circ_ORF_ID`
2. `search_ORF_IDs`
3. `circRNA_ID`
4. `gene`
5. `IRES_Up50_Center`
6. `IRES_Up150_Center`
7. `IRES_Down_Stop`
8. `m6A_Count`
9. `cORF_seq`
10. `cORF_len`
11. `cORF_map`
12. `peptide_coverage`
13. `cORF_bsj_position`
14. `MS_count`
15. `total_peptide_count`
16. `unique_peptide_count`
17. `peptide_seq`
18. `ms_sample_evidence`
19. `coding_function_score`

Optional column:

- `strand`

`strand` is included only when `eval` can read strand information from `circ_info_omit.gff` or a compatible circRNA annotation source.

### 5.2 Per-column meanings

| Column | Meaning |
| --- | --- |
| `circ_ORF_ID` | Primary ORF identifier used as the final source-level circRNA ORF record after mapping search-space ORFs back to their biological source ORF. |
| `search_ORF_IDs` | One or more comma-separated search-space ORF identifiers that were collapsed onto the same source ORF through `circRNA_ORF_mapping.tsv`. |
| `circRNA_ID` | Base circRNA identifier extracted from the ORF identifier after removing internal prefix tags such as `F_base_`, `R_exon_`, or similar source-mode markers. |
| `gene` | Gene label associated with the circRNA/ORF when available; missing values may appear as `none`, `NA`, `unknown`, or similar placeholders depending on input mode. |
| `strand` | Strand annotation for the circRNA, typically `+` or `-`, when the upstream ORF workflow provided usable annotation. |
| `IRES_Up50_Center` | DeepCIP-derived IRES-related score for the sequence window centered around the region 50 nt upstream of the candidate ORF start context. |
| `IRES_Up150_Center` | DeepCIP-derived IRES-related score for the sequence window centered farther upstream around the ORF start context. |
| `IRES_Down_Stop` | DeepCIP-derived IRES-related score for the sequence window anchored downstream toward the ORF stop-side context. |
| `m6A_Count` | Count of predicted m6A-positive windows associated with the ORF from bundled DeepCircM6A inference. |
| `cORF_seq` | Amino-acid sequence of the candidate circRNA ORF used for peptide mapping and ranking. |
| `cORF_len` | Length of `cORF_seq` after excluding the terminal stop marker from the amino-acid count. |
| `cORF_map` | Sequence-length map string showing BSJ positions and peptide-supported amino-acid positions on the cORF. |
| `peptide_coverage` | Fraction of amino-acid positions in the cORF covered by at least one retained peptide after linear-proteome filtering. |
| `cORF_bsj_position` | Internal BSJ position code copied from the ORF/mapping context; multiple BSJ sites are separated by `|`. |
| `MS_count` | Number of distinct retained MS sample IDs or evidence-source IDs supporting this ORF. |
| `total_peptide_count` | Total retained peptide-support count summed across all retained peptide entries after collapsing repeated sample-peptide observations. |
| `unique_peptide_count` | Number of distinct retained peptide sequences supporting this ORF. |
| `peptide_seq` | Semicolon-separated compact peptide summary showing peptide identity plus evidence counts. |
| `ms_sample_evidence` | Semicolon-separated `sample_or_source_id:count` summary of aggregated evidence counts per MS sample or source identifier. |
| `coding_function_score` | Composite ranking score used to sort circRNA ORFs by predicted coding/translation potential. Higher values indicate stronger integrated support. |

### 5.3 How to read `cORF_map`

`cORF_map` is a compact coverage-annotation string aligned to the amino-acid sequence.

Its character rules are:

- lowercase amino-acid letters: the residue is covered only by peptide evidence that does not cross the BSJ
- uppercase amino-acid letters: the residue is covered by at least one peptide that crosses the BSJ, which is stronger circRNA-specific support
- `x`: no retained peptide evidence covers this amino-acid position
- `|`: an inserted BSJ separator marking the BSJ site in the displayed map
- terminal `*`: retained only when the original ORF sequence itself contains a stop marker

Practical interpretation:

- a long uppercase block usually indicates high-value BSJ-spanning peptide support
- lowercase-only support still indicates translation evidence, but not direct BSJ crossing
- `x`-rich maps indicate sparse peptide support across the ORF

### 5.4 How to read `peptide_seq`

`peptide_seq` uses the format:

```text
PEPTIDE(ms_hit_count|total_count);another_peptide(ms_hit_count|total_count)
```

Rules:

- uppercase peptide entry: the peptide belongs to the BSJ-crossing peptide set for this ORF
- lowercase peptide entry: the peptide supports the ORF but does not cross the BSJ
- `ms_hit_count`: number of distinct retained MS sample IDs or evidence-source IDs supporting that peptide
- `total_count`: aggregated retained count for that peptide after collapsing repeated entries from the parsed MS table

Example:

```text
HSLTTLTEQWIILR(1|6);llteisslwdgcqk(1|5)
```

This means:

- `HSLTTLTEQWIILR` is a BSJ-crossing peptide, supported by 1 distinct sample/source ID with an aggregated retained count of 6
- `llteisslwdgcqk` is a non-BSJ-crossing peptide, supported by 1 distinct sample/source ID with an aggregated retained count of 5

### 5.5 How to read `cORF_bsj_position`

`cORF_bsj_position` is the internal BSJ location code inherited from the ORF header or ORF mapping context and is used by MStoCIRC2 to decide whether a peptide crosses the BSJ.

Important notes:

- a single value means one BSJ site is represented in the current ORF frame
- multiple values separated by `|` indicate repeated or multiple BSJ positions in the ORF representation
- decimal suffixes are preserved from the internal ORF encoding and should be treated as internal BSJ-position markers rather than as standalone biological scores

### 5.6 How `coding_function_score` is built

`coding_function_score` is a weighted ranking score rather than a direct probability.

It combines:

- IRES support
- m6A support
- peptide coverage on the ORF
- BSJ-crossing peptide evidence
- non-BSJ-crossing peptide evidence

In the current implementation, BSJ-crossing peptide evidence contributes more heavily than non-BSJ-crossing evidence, so ORFs with direct BSJ-spanning peptide support are intentionally prioritized.

## 6. DEA Module Outputs

Default location:

- direct `dea` run: the DEA output directory
- integrated `quant`: `<root>/dea/`

### 6.1 Core quantitative tables

| File or pattern | Description |
| --- | --- |
| `peptide_filter_report.tsv` | Row-level peptide filtering report recording which peptide entries were retained or removed and why. |
| `protein_quant.tsv` | Protein-level circRNA-aware quantitative matrix generated after peptide filtering and peptide-to-protein roll-up. |
| `rollup_summary.tsv` | Per-protein summary of retained peptide counts, circRNA flags, and roll-up metadata used to interpret the quantitative matrix. |
| `protein_quant_imputed.tsv` | Imputed protein-level matrix generated after missing-value handling according to the selected DEA strategy preset. |
| `dea_results.tsv` | Main differential-expression result table containing effect sizes, test statistics, P-values, adjusted P-values, and circRNA-related annotations. |

### 6.2 Visualization files

| File or pattern | Description |
| --- | --- |
| `volcano_plot.pdf`, `volcano_plot.png` | Volcano plots summarizing the differential-expression landscape of the current contrast in vector and raster formats. |
| `enrichment_<group>_GO_barplot.(pdf|png)` | GO enrichment bar plots for the proxy-gene set associated with circRNA candidates upregulated in the specified group. |
| `enrichment_<group>_GO_dotplot.(pdf|png)` | GO enrichment dot plots for the proxy-gene set associated with circRNA candidates upregulated in the specified group. |
| `enrichment_<group>_KEGG_barplot.(pdf|png)` | KEGG enrichment bar plots for the proxy-gene set associated with circRNA candidates upregulated in the specified group. |
| `enrichment_<group>_KEGG_dotplot.(pdf|png)` | KEGG enrichment dot plots for the proxy-gene set associated with circRNA candidates upregulated in the specified group. |

### 6.3 Correlation and enrichment tables

| File or pattern | Description |
| --- | --- |
| `correlation_<group>.tsv` | circRNA-mRNA correlation result table for the specified contrast side or upregulated group. |
| `proxy_genes_<group>.tsv` | Proxy mRNA gene list selected for the specified group before GO/KEGG enrichment analysis. |
| `enrichment_<group>_go_BP.tsv` | GO biological-process enrichment result table for the specified group. |
| `enrichment_<group>_go_CC.tsv` | GO cellular-component enrichment result table for the specified group. |
| `enrichment_<group>_go_MF.tsv` | GO molecular-function enrichment result table for the specified group. |
| `enrichment_<group>_kegg.tsv` | KEGG enrichment result table for the specified group. |

### 6.4 Metadata and conditional outputs

| File or pattern | Description |
| --- | --- |
| `run_metadata.json` | Compact run summary recording strategy, organism, contrast, filtering summary, volcano summary, enrichment summary, and total runtime. |
| `dea_skipped.txt` | Integrated `quant` writes this file instead of running DEA when fewer than two groups can be reconstructed from the manifest. |
| `.mstocirc2_runtime/dea/design.from_manifest.tsv` | Hidden integrated-workflow helper file containing the DEA design table rebuilt from the manifest when no DIA experiment annotation is available. |
| `.mstocirc2_runtime/dea/design.from_annotation.tsv` | Hidden integrated-workflow helper file containing the DEA design table rebuilt from `experiment_annotation.tsv` for DIA quant runs. |
